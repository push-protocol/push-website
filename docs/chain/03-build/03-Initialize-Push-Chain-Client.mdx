### Initialize Push Chain parameters

When initializing your Push Chain instance, you can customize the process using several parameters. Here's a breakdown:

| Param                 | Type                             | Default                                          | Remarks                                                                                                                                                                                          |
| --------------------- | -------------------------------- | ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| _`universalSigner`_   | `UniversalSigner`                | -                                                | Chain-agnostic signer responsible for signing transactions and messages.                                                                                                                         |
| `options.network`     | `PUSH_NETWORK`                        | `PushChain.CONSTANTS.PUSH_NETWORK.TESTNET_DONUT` | Push Chain environment. Options: `MAINNET`, `TESTNET_DONUT`, `LOCALNET`                                                                                                                                |
| `options.rpcUrls`     | `Partial<Record<CHAIN, string>>` | `{}`                                             | Custom RPC URLs mapped by chain IDs. If not provided, the default RPC URLs for the network will be used. Example: `rpcUrls: {[CHAIN.ETHEREUM_SEPOLIA]: 'https://sepolia.infura.io/v3/your-api-key', [CHAIN.SOLANA_DEVNET]: 'https://api.devnet.solana.com'}` |
| `options.printTraces` | `boolean`                        | `false`                                          | When true, console logs the internal trace logs for debugging requests to nodes                                                                                                                  |

#### `NETWORK` Values

The `NETWORK` enum specifies the Push Chain environment the SDK will connect to. It determines the RPC endpoints, contract addresses, and network context for interacting with Push Chain.

| Value                        | Description                   |
| ---------------------------- | ----------------------------- |
| `MAINNET`                    | Production environment        |
| `TESTNET_DONUT` or `TESTNET` | Testing environment           |
| `LOCALNET`                   | Local development environment |

#### `CHAIN` Values

The `CHAIN` enum defines all supported blockchain networks that can be used with Push Chain.

| Value                                  | Network Type | Description                          |
| -------------------------------------- | ------------ | ------------------------------------ |
| `PUSH_MAINNET`                         | Push Chain   | Push Chain mainnet                   |
| `PUSH_TESTNET_DONUT` or `PUSH_TESTNET` | Push Chain   | Push Chain testnet                   |
| `PUSH_LOCALNET`                        | Push Chain   | Push Chain local development network |
| `ETHEREUM_MAINNET`                     | Ethereum     | Ethereum mainnet                     |
| `ETHEREUM_SEPOLIA`                     | Ethereum     | Ethereum testnet (Sepolia)           |
| `SOLANA_MAINNET`                       | Solana       | Solana mainnet                       |
| `SOLANA_DEVNET`                        | Solana       | Solana development network           |

## Example

Here is an example of how to initialize the Push Chain SDK with a universal signer and a custom RPC URL.

<Tabs groupId="initialize-push-chain-client">
  <TabItem value="ethers_v6" label="Ethers (v6)" default>
<NodeJSVirtualIDE repo={{title: "Open in Github", url: "https://github.com/push-protocol/push-chain-sdk"}}>

  {`
  import { PushChain } from '@pushchain/core'
  import { ethers } from 'ethers'

  async function main() {
      console.log('Creating Universal Signer - Ethers V6')

      // Create random wallet
      const wallet = ethers.Wallet.createRandom()

      // Set up provider connected to Ethereum Sepolia Testnet
      const provider = new ethers.JsonRpcProvider('https://gateway.tenderly.co/public/sepolia')
      const signer = wallet.connect(provider)

      // Convert ethers signer to Universal Signer
      const universalSigner = await PushChain.utils.signer.toUniversal(signer)
      console.log('🔑 Got universal signer')

      // Initialize Push Chain SDK
      const pushChainClient = await PushChain.initialize(universalSigner, {
          network: PushChain.CONSTANTS.PUSH_NETWORK.TESTNET_DONUT,
      })
      console.log('🔑 Got push chain client')
      console.log(JSON.stringify(pushChainClient))
  }

  main().catch(console.error)
  `}

## Get Push Chain Client UEA and Universal Origin Account

The Push Chain Client provides access to two important accounts: the UEA (Universal Executor Account) and the Universal Origin Account. The UEA is the smart contract account on Push Chain that executes transactions when user is using fee abstraction, while the Universal Origin Account represents the Universal Account of the user's wallet on the blockchain network where the user's wallet originates from (e.g., Ethereum Sepolia, Solana Devnet, etc.).

```typescript
// UEA Address
const ueaAddress = pushChainClient.universal.account

// Origin Universal Account
const originAccount = pushChainClient.universal.origin
```

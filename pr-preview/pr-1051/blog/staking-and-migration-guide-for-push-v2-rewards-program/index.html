How to Claim Rewardsâ€‹

Claiming rewards for both the pools is fairly similar.

  1. Go to the respective pool on Push Dapp Yield Farming V2 page.
  2. Click on the Claim Rewards button of the PUSH Fee Staking Pool or UNI-V2 Staking Pool.
  3. Confirm the transaction and rewards will be transferred to your wallet.

However, there are a couple of considerations to keep in mind related when harvesting rewards.

ðŸ”´ Harvesting in PUSH Fee Staking Poolâ€‹

The very first claim reward procedure for this pool shall include two transactions:

  • 1st Transaction is to Set Holder Delegation for the Push Core Contract address.
  • 2nd transaction is to trigger the primary harvest function of the Push core contract to release the rewards.
What exactly is Setting Holder Delegation?
  • The $PUSH token has a time-weightage feature within itself which allows the contract to calculate the holder weight of a $PUSH token holder based on how long he/she has been holding the token.
  • During the harvest, the core contract is designed to be capable of updating the holder weight of the caller to the latest block.
  • Therefore, this set holder delegationtransaction approves the main Push Core contract to be capable of updating holder weight whenever needed.
  • It should be noted that this permission can be revoked anytime by the user.

Note: The setting holder delegation is a one-time process and only takes place during the very first instance of claiming rewards.

  • Go to the UNI-V2 LP Pool on Push Dapp Yield Farming V2 page.
  • Click on the Claim Rewards button of the UNI-V2 Staking Pool.
  • Confirm the transaction and rewards will be transferred to your wallet.

If thatâ€™s the case for you, the migration feature is specifically designed to provide you with a smooth and easy procedure to shift from the old staking pool to the new v2 staking pool.

Migration from old to new pools requires 3 simple transactions:

  1. Unstaking from old contracts: This transaction withdraws your staked tokens from the old contracts.
  2. Approval of tokens: This ensures you approve the new staking pool contracts.
  3. Staking into new pool: The final transaction ensures a successful deposit of your tokens to the new staking pools.

In order to perform the migration procedure, please follow the steps below:

  2. Click on Migrate to PUSH Fee Pool button
  3. After the above-mentioned 3 transactions, your staked tokens will be migrated from the previous $PUSH Token staking contract to the new $PUSH Fee Staking Pool (which essentially is the main Push Core contract itself).
